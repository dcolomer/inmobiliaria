Un  diseño por capas presenta muchos beneficios:

·         Un bajo acoplamiento y una alta cohesión. Un sistema en el que sus módulos tengan poca dependencia entre ellos y en el que el contenido de cada módulo sea muy pertinente, es un sistema fácil de mantener, ya que es fácil de entender.

·         Fácil sustitución de la tecnología y/o técnica empleada en el diseño de cada nivel. Por ejemplo, en la capa de persistencia utilizaremos para la implementación JDBC puro, pero nada nos impedirá más adelante sustituir tal clase por una implementación con Hibernate.

En una primera fase la aplicación no será distribuida, es decir, todas las capas se deberán ejecutar en la misma MVJ (en un único ordenador, a efectos prácticos).

En una segunda fase, ya con la aplicación totalmente operativa, utilizaremos la tecnología RMI de Java o Servicios Web para dividir la aplicación en dos partes:

·         La aplicación cliente, formada por la capa de presentación

·         La aplicación servidora, formada por la capa de negocio y la de persistencia

Con lo cual tendremos una aplicación servidora dando servicio a varias aplicaciones cliente.

 

Capa de Persistencia

Comenzaremos la práctica trabajando en este nivel. Para ello crearemos las clases de acceso a datos y comprobaremos su correcto funcionamiento antes de pasar a siguiente nivel. Procediendo de esta forma nos aseguramos que lo que vamos construyendo es sólido.

Es una práctica habitual diseñar la capa de persistencia utilizando el patrón DAO (Data Access Object), el cual abstrae y encapsula toda la lógica de acceso a datos, es decir, el resto de capas de la aplicación no van a saber absolutamente nada respecto a la tecnología utilizada para el acceso a datos (JDBC, Hibernate, etc). Si nos fijamos en la figura anterior, apreciaremos que la capa de negocio es el único nivel que conecta con el nivel de persistencia, por lo que un componente de negocio, sencillamente enviará un mensaje a un servicio ofrecido por un componente DAO para que éste, de manera transparente, realice el trabajo (Una clase de negocio tendrá un atributo de la clase DAO –mecanismo de composición- por este motivo podrá enviarle mensajes, es decir, podrá utilizarlo).

Por otra parte diseñaremos orientado a las interfaces, dado que conocemos las ventajas de disponer de una interfaz donde definir nuestros métodos y la posibilidad de tener más de una clase para implementar tal interfaz (es bueno no atarnos a una única implementación).

Importante: Cada uno de los métodos DAO debe ser una operación atómica, esto es, no transaccional. Será la capa de negocio la que dispondrá de métodos más sofisticados que considerarán agrupar en una transacción diversas operaciones DAO. Por ejemplo, en nuestra aplicación, la reserva de un piso será una operación de la capa de negocio que incluirá en una transacción varias operaciones DAO:

·         Comprobar disponibilidad de piso para las fechas especificadas

·         En caso afirmativo:

o   Generar nuevo pedido

o   Generar nueva factura

 

Propuesta de clases e interfaces para la capa de persistencia

La siguiente figura muestra un posible diseño para esta capa:

Analicemos el diagrama anterior:

Clase GestorPersistencia/Configuracion: Implementa el patrón Singleton para obtener un único objeto java.sql.Connection/DataSource de la base de datos especificada.

Clase AbstractDao: Clase abstracta que se contiene atributos y métodos comunes para sus clases hijas. Aquí definimos un objeto Connection como protected, así podrá ser utilizado en las subclases como si se hubiera definido en ellas. Más adelante también definiremos un objeto Log que será utilizado en las clases hijas para anotar los sucesos más relevantes de las operaciones de acceso a datos. También implementaremos cualquier método que pudiera duplicarse en las clases hijas, como podría ser el caso de un método que retorne el valor de la última clase primaria generada automáticamente por el SGBD.

Interfaz DaoGestion: Definición de los métodos relativos a las operaciones CRUD de la aplicación (clientes, propietarios, pisos).

Clase DaoGestionImpl: Implementación de la interfaz DaoGestion.

Interfaz DaoPagos: Definición de los métodos relativos a las operaciones de pagos y cobros de la aplicación (nuevaFactura, actualizarPedido, getPisosDisponibles, cerrarCaja, etc).

Clase DaoPagosImpl: Implementación de la interfaz DaoPagos.

Interfaz DaoInmobiliaria: Interfaz envoltorio que extiende a la interfaz DaoGestion y DaoPagos. Disponer de una interfaz como esta permite a la capa de negocio utilizar cualquier método de acceso a datos de una manera muy cómoda.

Clase DaoInmobiliariaImpl: Implementación de la interfaz DaoInmobiliaria. Es una clase envoltorio que implementa todos los métodos de las interfaces DaoGestion y DaoPagos realizando una sencilla llamada a los métodos homónimos de las clases de implementación de tales interfaces.

package dao;

 

import java.util.List;

import entidades.Cliente;

import entidades.Piso;

import entidades.Propietario;

 

public interface DaoGestion {

 

      /*

       * CRUD Cliente

       */

      int grabarCliente(Cliente cliente);

      List<Cliente> getClientes();

      int eliminarCliente(String nif);

      Cliente getCliente(String nif);

     

      /*

       * CRUD Propietario

       */

      int grabarPropietario(Propietario propietario);     

      List<Propietario> getPropietarios();

      int eliminarPropietario(String nif);

      Propietario getPropietario(String nif);

     

      /*

       * CRUD Piso

       */

      int grabarPiso(Piso piso);

      List<Piso> getPisos();

      int eliminarPiso(long n_piso);

      Piso getPiso(long n_piso);

      List<Piso> getPisosPropietario(String nif);

     

}

Nota1: Se utiliza como framework de acceso a datos las DBUtils de Apache Commons.

Nota2: No se proporciona el método para todos los métodos, sino únicamente para los métodos grabarCliente(Cliente) y getCliente(String nif).

 

package dao;

 

import java.sql.SQLException;

import org.apache.commons.dbutils.QueryRunner;

import org.apache.commons.dbutils.handlers.BeanListHandler;

import org.apache.commons.dbutils.handlers.ScalarHandler;

 

import entidades.Cliente;

import entidades.Piso;

import entidades.Propietario;

 

public class DaoGestionImpl extends AbstractDao implements DaoGestion {

     

      public DaoGestionImpl() {

            System.out.println("OBJETO DaoGestionImpl CONSTRUIDO");

      }

 

      // **********************************************************************

      // ************************** Cliente ***********************************

      // **********************************************************************

     

      /*

       * Si el cliente existe lo modificamos.

       * Si no existe lo damos de alta

       */

      @Override

      public int grabarCliente(Cliente cliente) {

                 

            Cliente clienteRecuperado=getCliente(cliente.getNif_cli());

           

            if (clienteRecuperado!=null && clienteRecuperado.equals(cliente)) {

                  return modificarCliente(cliente);             

            } else

                  return nuevoCliente(cliente);

      }

 

/*

       * Recupera un cliente de la BD a partir del NIF

       */

      @SuppressWarnings("unchecked")

      @Override

      public Cliente getCliente(String nif) {

                       

            Cliente cliente=null;

            String SQL_CLIENTE_POR_NIF="SELECT * FROM CLI WHERE nif_cli=?";       

           

        QueryRunner qr = new QueryRunner(getDataSource());

        Object[] params = new Object[]{nif};

       

        @SuppressWarnings("rawtypes")

            ResultSetHandler<Cliente> rsh = new BeanHandler(Cliente.class);

       

        try {               

                 

            if (params[0]!=null) // Si se ha pasado el NIF al método

                  cliente=qr.query(SQL_CLIENTE_POR_NIF, rsh, params);

           

            LOG.info("Cliente recuperado desde NIF: "+cliente);

           

            closeConn();

           

            } catch (SQLException e) {

                  LOG.error("Error de JDBC al recuperar el cliente con NIF: "+nif, e);

                  e.printStackTrace();

            }

           

            return cliente;        

      }

/*

       * método privado para modificar un Cliente en la BD

       */

      private int modificarCliente(Cliente cliente) {

            int actualizadas=0;

            String SQL_MODIFICAR_CLIENTE="UPDATE CLI SET nombre=?, apel=? WHERE nif_cli=?";

           

            QueryRunner run = new QueryRunner(getDataSource());

           

            try   {                     

                actualizadas = run.update(SQL_MODIFICAR_CLIENTE,

                        cliente.getNombre(), cliente.getApel(), cliente.getNif_cli());

               

                LOG.info("cliente modificado: "+cliente );

               

                closeConn();

            }

            catch(SQLException e) {

             LOG.error("Error de JDBC al modificar el cliente: "+cliente, e );

                  e.printStackTrace();

            }

           

            return actualizadas;

      }

/*

       * método privado para dar de alta un Cliente en la BD

       */

      private int nuevoCliente(Cliente cliente) {

            int insertadas=0;

            String SQL_NUEVO_CLIENTE="INSERT INTO CLI (nif_cli, nombre, apel) VALUES (?,?,?)";

           

            QueryRunner run = new QueryRunner(getDataSource());

           

            try   {

                   /*

                    * Ejecutar la instrucción de actualización y retornar

                    * el número de filas afectadas

                    *

                    *  Importante: En el método update() el segundo parámetro

                    *  es un varargs (y utiliza autoboxing si se requiere) por

                    *  lo que siempre podremos especificar el numero de campos

                    *  que necesitemos

                    * 

                    */

                                

                insertadas = run.update(SQL_NUEVO_CLIENTE,

                        cliente.getNif_cli(), cliente.getNombre(), cliente.getApel());

 

                LOG.info("Cliente añadido: "+cliente );

               

                closeConn();

            }

            catch(SQLException e) {

                  LOG.error("Error de JDBC al crear el cliente: "+cliente, e );

                  e.printStackTrace();

            }          

            return insertadas;

      }
